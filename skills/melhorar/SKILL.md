# Melhorar Roteiro

## Trigger

Após crítica de roteiro, ou quando o usuário pede para melhorar um texto.

## Fluxo

### 1. Reunir inputs
- Roteiro original
- Crítica/análise (se houver)
- Pedidos do usuário

### 2. Perguntar se necessário
- "A crítica sugeriu [X]. Quer que eu siga nessa direção?"
- "Posso ampliar o tema para [Y] — funciona pro seu público?"
- "Quer manter o tom ou posso mudar [aspecto]?"

### 3. Reescrever (agente: melhorador)
- Preservar pontos fortes identificados
- Resolver problemas na ordem de prioridade
- Aplicar princípios ausentes de forma orgânica
- Manter a voz/tom do original

### 4. Entregar
- Versão melhorada completa
- Lista curta: "O que mudei e por quê"
- Sugestão de nova crítica para validar

### 5. Iteração
Se o usuário quiser mais ajustes, repetir o ciclo.

## Regras

- Nunca destruir o que já funciona
- Mudanças cirúrgicas > reescrita total
- Perguntar antes de decisões criativas grandes
- Manter regras universais (não inventar, fontes verificáveis)
