# Crítica de Roteiro

## Trigger

Usuário pede para criticar/avaliar um roteiro (colado ou em arquivo).

## Fluxo

### 1. Receber roteiro
- Aceitar texto colado diretamente
- Ou path para arquivo .md no vault
- Perguntar se necessário: "Qual é o público/nicho desse conteúdo?"

### 2. Análise de princípios (agente: analista-de-principios)
- Anotar frase a frase quais princípios estão presentes
- Identificar quais DEVERIAM estar e não estão
- Ser honesto — não forçar princípios onde não existem

### 3. Crítica honesta (agente: critico)
Avaliar:
- Hook: funciona ou não? Por quê?
- Retenção: onde o público desistiria?
- Debate mental: provoca pensamento ativo?
- Redundância: repete demais?
- Engajamento externo: geraria compartilhamento/comentário?
- Moral: conecta com a vida do público?

### 4. Output estruturado

```markdown
## Diagnóstico

**Ideia central:** [qual ideia o vídeo vende]
**Tamanho do tema:** [universal / bolha / nicho restrito]
**O que está carregando o roteiro:** [ponto(s) forte(s)]

## Hook
- Princípios presentes: [lista]
- Avaliação: [funciona/não funciona + por quê]
- Nota: [1-10]

## Princípios aplicados
| Princípio | Presente? | Onde | Intensidade |
|-----------|-----------|------|-------------|
| Tema universal | ✅/❌ | frase X | forte/fraco |
| Familiaridade | ✅/❌ | ... | ... |
| Conflito/Mudança | ✅/❌ | ... | ... |
| Curiosidade | ✅/❌ | ... | ... |
| Debate mental | ✅/❌ | ... | ... |
| Contraste | ✅/❌ | ... | ... |
| Aha | ✅/❌ | ... | ... |
| Moral | ✅/❌ | ... | ... |

## Problemas identificados
1. [Problema mais grave primeiro]
2. [...]

## O que poderia ser amplificado
1. [Sugestão concreta]
2. [...]

## Veredicto
- Potencial de retenção: [baixo/médio/alto]
- Potencial de viralização: [baixo/médio/alto]
```

### 5. Oferecer próximos passos

> "Quer que eu melhore esse roteiro aplicando as sugestões?"

## Regras

- Honestidade absoluta — não elogio vazio, não crítica destrutiva
- Específico sempre — "onde" e "por quê", nunca genérico
- Priorizar — maior problema primeiro
- Reconhecer o que funciona — não é só lista de defeitos
- Seguir a regra do framework: "O que está carregando e o que pode amplificar?"
